## 三、视觉与布局（严格对齐）

### 1. 整体风格

- 极简（Minimal）
- 高对比度
- 大字号排版
- 留白非常克制
- 没有多余装饰元素

### 2. 颜色

- 背景：接近纯黑 / 深灰
- 文字：纯白 / 浅灰
- 强调色：极少使用（仅用于 hover / 细节）

### 3. 字体

- 使用现代无衬线字体（如 Inter / SF Pro / 类似）
- Hero 标题为超大字号
- 全大写字母
- letter-spacing 明显
- 字重偏粗，但不夸张

## 四、Hero Section（最重要）

必须实现以下细节：

- 高度：100vh
- 内容垂直 + 水平居中
- 主标题文本类似：
  “FULL-STACK DEVELOPER”
- 副标题为较小字号
- 所有文字是「视觉中心」

### Hero 动画（必须）

- 页面加载时：
  - 主标题淡入 + 位移（y 轴）
  - 副标题延迟出现
- 使用 GSAP timeline
- ease 必须自然（如 power3.out）

## 五、滚动与动画逻辑（核心）

### 1. 滚动方式

- 使用 Lenis 或等效 smooth scroll
- 滚动非常顺滑
- 没有原生滚动的顿挫感

### 2. ScrollTrigger 行为

- 内容不是一次性出现
- 每个 section 都有：
  - fade
  - translate
  - 或轻微 scale
- 动画与滚动强绑定
- scrub 使用得当（不是所有动画都 scrub）

### 3. 视差感（如果原站存在）

- 前景与背景滚动速度不同
- 文本与容器存在轻微速差

## 六、Hover 与交互细节（不能忽略）

### 鼠标 Hover

- 文本 hover：
  - 颜色轻微变化
  - 或 underline 动画
- 卡片 hover：
  - scale 轻微放大（1.02 ~ 1.05）
  - 阴影或亮度变化
- 所有 hover 都是「克制的」

### 点击

- 内部 anchor 平滑滚动
- 没有突兀跳转
